## Objective

Simulate a brute-force attack against a Windows system and investigate whether subsequent successful logon events indicate a compromised account or normal system behavior.

## Environment

| Component | Role | IP Address |
| --- | --- | --- |
| Wazuh SIEM | Log analysis | 192.168.56.10 |
| Windows Victim | Target | 192.168.56.40 |
| Kali Linux | Attacker | 192.168.56.30 |

## Pre-Conditions

- Windows agent active in Wazuh
- Network connectivity verified
- Valid user account (`victim`) created

## Attack Simulation

A brute-force attack was performed from Kali Linux targeting the Windows machine using SMB authentication.

Multiple incorrect password attempts were generated to simulate credential guessing.

![image.png](attachment:411de445-357e-4a23-a390-57e92350f868:image.png)

## Detection

Wazuh detected:

- Multiple failed login attempts (Event ID 4625)
- Account lockout due to repeated failures
- Subsequent successful logon events (Event ID 4624)

![image.png](attachment:0e3f4f2c-394f-4646-a0a7-2f04c2c3c573:image.png)

---

## Observed Events

- **Failed Logons:** Event ID 4625
- **Account Lockout:** Rule ID 60115
- **Successful Logons:** Event ID 4624

## Analysis

Initial observation suggested a potential account compromise due to:

```
Multiple failed attempts → Account lockout → Successful logon
```

However, further investigation revealed:

- Successful logons had **Logon Type 5 (Service Logon)**
- No evidence of attacker IP (192.168.56.30)
- Events were generated by system processes

![image.png](attachment:1e437ab2-2bce-4a82-9b83-44ee50166a8d:image.png)

## Key Findings

- The brute-force attack was successfully detected
- The account was locked due to multiple failed attempts
- Successful logon events were **not related to the attacker**
- No evidence of unauthorized access was found

## MITRE ATT&CK Mapping

- **Tactic:** Credential Access
- **Technique:** T1110 — Brute Force

## Conclusion

Although successful logon events were observed, they were determined to be system-generated and not indicative of a compromised account.

This highlights the importance of validating context before concluding a security incident.

## Recommendations

- Implement account lockout policies
- Monitor repeated authentication failures
- Correlate events before escalating incidents
- Validate logon types and source IPs during investigations
